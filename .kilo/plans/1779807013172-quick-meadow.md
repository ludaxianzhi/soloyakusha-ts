# 目标：重构 targetGroups 填充逻辑

## 现状

- `targetGroups`（`target_groups_json`）记录每行原文的多个译文历史版本
- 目前仅导入/解析时初次写入，后续流程未主动维护
- `replaceFragmentTranslations` 仅 preserve 旧值，不添加新译文

## 改造目标

每次翻译步骤完成时：
1. 清空 fragment 的 `targetGroups`（从 `[]` 开始）
2. 将所有产生/编辑译文的内部步骤输出按顺序追加到 `targetGroups`
   - 默认翻译器（1 步）：`[["step1_line0"], ["step1_line1"], ...]`
   - 风格迁移翻译器（2 步：translator + styleTransfer）：`[["step1_line0","step2_line0"], ...]`
3. `translation` 行逻辑保持不变

## 修改清单

### 1. `TranslationProcessorResult` 新增字段

**文件**: `src/project/processing/translation-processor.ts`

添加可选字段：
```typescript
/** 各内部步骤按顺序输出的逐行译文数组。用于填充 targetGroups。 */
stepTranslations?: string[][];
```

**文件**: `src/project/processing/style-transfer-translation-processor.ts`

- return 中追加 `stepTranslations: [initialTranslations, finalTranslations]`

## 边界情况

- `stepTranslations` 为 undefined/空数组时保持现有行为（不更新 targetGroups）
- 非 finalStepId 的步骤（如 style-transfer 的 analyzer）不更新 translation 行，也不更新 targetGroups
- 取消/失败的结果不走 targetGroups 更新
- `clearAllTranslations` / `clearChapterTranslations` 已通过 `resetFragmentToUntranslated` 将 targetGroups 置空，不需要额外改动
